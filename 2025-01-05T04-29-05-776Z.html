Dependency Types in package.json

What are all the dependency types available inside a package.json file? Just name them

Here are the main dependency types in a package.json file:

  1. dependencies
  2. devDependencies
  3. peerDependencies
  4. optionalDependencies
  5. bundledDependencies (or bundleDependencies)

Here’s a detailed explanation of the dependency types in a package.json file, followed by a comparison:

1. dependencies

  • Purpose: These are the packages required for your application to run in production. They are essential for the core functionality of your app.
  • Installed With: npm install or npm install <package>
  • Example: "express": "^4.18.2"
  • Use Case: Packages your app depends on to function, such as frameworks, libraries, or utilities (e.g., react, express).

2. devDependencies

  • Purpose: These are the packages used only during development, testing, or building your app. They are not included in the production deployment.
  • Installed With: npm install --save-dev or npm i -D
  • Example: "eslint": "^8.50.0"
  • Use Case: Tools like linters, test frameworks, bundlers, or transpilers (e.g., jest, webpack, eslint).

3. peerDependencies

  • Purpose: These specify that your package is compatible with a specific version of another package but does not install it automatically. They let the consuming app ensure compatibility with its own dependencies.
  • Installed With: Must be installed explicitly by the user.
  • Example: "react": ">=17.0.0"
  • Use Case: Libraries or plugins that work with a specific host framework or library (e.g., a React plugin requiring react).

4. optionalDependencies

  • Purpose: These are dependencies that your app can function without, but they enhance functionality if present. If installation fails, the error is ignored.
  • Installed With: npm install (like dependencies)
  • Example: "fsevents": "^2.3.2"
  • Use Case: Packages for additional features that may not work on all platforms (e.g., fsevents for file watching on macOS).

5. bundledDependencies (or bundleDependencies)

  • Purpose: These are dependencies that are included in the package itself when it is published. This ensures consumers of your package don’t need to install them separately.
  • Installed With: They are bundled during npm pack or npm publish.
  • Example: [ "lodash", "axios" ]
  • Use Case: Ensuring specific versions of dependencies are always included, regardless of the user's environment.

Comparison of Dependency Types

Type Installed in Production? Installed in Development? Managed Automatically? Purpose
dependencies ✅ Yes ✅ Yes ✅ Yes Core functionality for production.
devDependencies ❌ No ✅ Yes ✅ Yes Tools for development/testing.
peerDependencies ⚠️ No (manual install) ⚠️ No (manual install) ❌ No Compatibility with a host package.
optionalDependencies ✅ Yes ✅ Yes ✅ Yes Enhance functionality; safe to fail install.
bundledDependencies ✅ Yes ✅ Yes ✅ Yes (when published) Guarantee specific packages are bundled.

Key Differences:

  1. Production vs. Development:

    • dependencies are for production; devDependencies are for development only.

  2. Compatibility:

    • peerDependencies don’t install automatically and are used to specify compatibility, unlike dependencies.

  3. Fault Tolerance:

    • optionalDependencies ignore installation errors, unlike others.

  4. Bundling:

    • bundledDependencies are packaged directly with your app, whereas others are fetched from the registry during installation.

How can I change a dependency that I have already installed into a dev dependency?

To change an already installed dependency into a devDependency in a Node.js project, follow these steps:

Method 1: Reinstall the Dependency as a Dev Dependency

  1. Uninstall the Dependency:
   npm uninstall <package-name>

This removes the dependency from both node_modules and your package.json file.

  1. Reinstall it as a Dev Dependency:
   npm install --save-dev <package-name>

This will add the dependency under the devDependencies section in your package.json.

Method 2: Edit the package.json Directly

  1. Open the package.json file in a text editor.
  2. Locate the dependency under the dependencies section.
  3. Cut the dependency entry (e.g., "eslint": "^8.50.0") and paste it under the devDependencies section.
  4. Save the file.
  5. Run the following command to sync your package.json with the node_modules: 
   npm install

Method 3: Use npm install with --save-dev Flag

You can install the same dependency with the --save-dev flag, and it will automatically move to devDependencies. For example: 

   npm install --save-dev <package-name>

This ensures the dependency is placed in the correct section, even if it was already installed.

Verify Changes

After making changes, confirm the dependency has been moved to devDependencies:

  1. Check the package.json file to ensure the dependency is listed under devDependencies.
  2. Run npm ls <package-name> to verify the package is still installed.
